## 详细方案

### 错误体验优化（P0）
- **Toast** — 简短友好提示（"请求失败，请稍后重试"）
- **详情** — 点击展开完整错误信息（给技术用户）
- **操作按钮** — 重试/切换 Provider/查看文档
- 技术实现：统一 ErrorBoundary + 错误码映射表 + Toast 组件

### 移动端优化（P0）
- 当前 Admin 页面在手机上体验不好
- 关键操作（查看用量、切换 Provider、查看错误）必须移动端可用
- 用 Bottom Sheet 替代 Modal，更适合单手操作
- 技术实现：CSS Grid + Bottom Sheet（radix-ui/dialog）

### 批量写入优化（P0）
- 当前每次请求都写 KV（用量计数），月 50 万配额紧张
- 方案：内存缓冲 + 定时 flush（每 60 秒或满 100 条）
- 风险：进程重启会丢失未 flush 的数据（可接受，最多丢 60 秒）
- 技术实现：内存 ring buffer + setInterval flush + graceful shutdown

### 智能路由（P1，差异化核心）
- 延迟探测：定期 ping 各 Provider 端点
- 成本计算：基于 token 价格和 Provider 定价
- 可用性监控：基于 Cron 巡检结果
- 路由策略：用户可配置权重（延迟/成本/可用性）
- 切换逻辑：当 Provider 连续 3 次失败 → 自动切换 → 恢复后自动切回
- 技术实现：Edge Runtime 内存路由表 + KV 持久化 + Cron 探测
- 设计：reactflow 流程图编辑器 + 实时状态指示（绿/黄/红）

### API Key 安全增强（P1）
- AES-256-GCM 加密，master key 在 Vercel 环境变量
- Key 轮换：检测到 401/403 → 标记 Key 为 "needs_rotation" → 前端提醒
- 已有 Key 需要迁移（写一个迁移脚本）

## 核心结论

> **AI Relay 的差异化在于智能路由 + 轻量部署。**
> v2.4 聚焦体验打磨，v2.5 聚焦核心功能，v3.0 聚焦增长和变现。

## 设计原则确认

1. **移动端优先** — 所有新功能先考虑移动端体验
2. **渐进式披露** — 高级功能按需展开，不干扰新手
3. **一致性** — 统一使用 shadcn/ui 组件库
4. **可访问性** — 颜色对比度、键盘导航、屏幕阅读器
