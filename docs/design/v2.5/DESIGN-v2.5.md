# AI Relay v2.5 设计规范

> 版本：v1.0 · 设计师：像素姐 · 日期：2026-05-27
> 范围：智能路由可视化 + API Key 安全管理

## 1. 设计目标

AI Relay v2.5 让中继站从「配置工具」进化为「智能路由管控平台」。

核心体验目标：
- 用户 10 秒内理解当前路由拓扑和 Provider 健康状态。
- 一键切换路由策略（延迟优先 / 成本优先 / 可用性优先）。
- API Key 安全状态一目了然，轮换操作 3 步内完成。

## 2. 信息架构

沿用现有 Admin 顶部 Header + Tab 结构，新增两个 Tab：

```text
Admin
├── Setup / 快速开始
├── Overview / 运行概览
├── Keys / 密钥管理
├── Models / 模型别名
├── Priority / 优先级规则
├── Health / 健康状态
├── Routing / 智能路由    ← 新增 (v2.5)
├── Security / Key安全    ← 新增 (v2.5)
├── Usage / 用量报告
├── Logs / 请求日志
├── Tools / 辅助工具
└── Webhooks / 通知设置
```

## 3. 设计语言复用

继续使用 dark glassmorphism 体系，不引入新品牌色。

关键 Token（同 v2.1）：
- 背景：`radial-gradient(circle at top, #1e293b, #09090b)`
- 面板：`rgba(30, 41, 59, 0.45)` + `backdrop-filter: blur(12px)`
- 边框：`rgba(255,255,255,0.08)`
- 主文字：`#e5e7eb`，次文字：`#9ca3af`
- Active Tab：蓝紫半透明渐变 + `#60a5fa`
- 健康绿：`#34d399`，警告黄：`#fbbf24`，错误红：`#f87171`
- 新增色 token — 安全紫：`#a78bfa`（加密状态标识）

布局：桌面最大宽度 `1200px`（路由编辑器需要更宽画布），移动端 Tab 横向滚动。

## 4. 智能路由可视化（Routing Tab）

### 4.1 页面结构

分为上下两区：
- **上区（60% 视口）**：路由流程图编辑器（reactflow canvas）
- **下区（40% 视口）**：路由策略配置面板

### 4.2 路由流程图编辑器

采用 reactflow 库实现节点+连线的可视化拓扑。

**节点类型：**

1. **入口节点**（Request In）
   - 固定在画布左侧
   - 显示：请求入口图标 + "Incoming Request"
   - 底部小字：当前 QPS

2. **策略节点**（Routing Strategy）
   - 位于中间区域
   - 显示：策略名称（延迟优先 / 成本优先 / 可用性优先）
   - 可点击切换策略类型
   - 图标随策略变化：⚡延迟 / 💰成本 / 🛡️可用性

3. **Provider 节点**
   - 位于右侧
   - 显示内容：
     - Provider 名称 + 图标
     - 健康状态灯：绿色(Healthy) / 黄色(Degraded) / 红色(Down) / 灰色(Unknown)
     - 当前延迟（ms）
     - 可用 Key 数 / 总 Key 数
     - 月请求量（mini sparkline）
   - 交互：
     - 单击：展开详情侧边栏
     - 拖拽：调整节点位置
     - 右键菜单：启用/禁用、查看详情、移除

4. **Fallback 节点**（虚线边框）
   - 表示降级路径
   - 连线为虚线，标签 "fallback"

**连线样式：**
- 主路径：渐变蓝紫实线 + 流动动画（表示请求流向）
- Fallback 路径：灰色虚线
- 禁用路径：暗灰实线，无动画
- 连线上显示：请求占比百分比

**画布操作：**
- 鼠标滚轮缩放
- 拖拽平移
- 小地图（右下角）
- 自动布局按钮（重置为最优布局）

### 4.3 路由策略配置面板

位于编辑器下方，使用 Tab 切换三种策略：

**延迟优先（Latency First）：**
- 滑块：最大可接受延迟（100ms ~ 2000ms）
- 开关：延迟探测频率（10s / 30s / 60s）
- 显示：当前各 Provider 平均延迟排名

**成本优先（Cost First）：**
- 每 Provider 的成本权重输入（$/1M tokens）
- 预算上限设置（月度 / 日度）
- 预算接近时的警告阈值（80% / 90%）

**可用性优先（Availability First）：**
- 健康检查间隔设置
- 故障转移阈值（连续 N 次失败后切换）
- 自动恢复设置（检测到恢复后自动切回）

**通用设置：**
- Sticky Session 开关（同一客户端固定 Provider）
- 超时设置（单 Provider 超时时间）
- 重试次数

### 4.4 实时状态叠加

编辑器右上角浮动面板显示：
- 当前活跃路由：`OpenAI → GPT-4o (72%) | Anthropic → Claude (28%)`
- 最近 1 分钟切换次数
- 全局健康状态汇总

### 4.5 移动端降级

移动端不显示 reactflow 编辑器，改为：
- 路由拓扑简化为垂直列表卡片
- 每个 Provider 一张卡片，显示状态和指标
- 策略配置使用全屏 Modal
- 可上下拖拽排序 Provider 优先级

## 5. API Key 安全管理（Security Tab）

### 5.1 页面结构

分为三个区域：
- **安全概览卡片区**（顶部）
- **Key 列表区**（主体）
- **轮换日志区**（底部折叠）

### 5.2 安全概览卡片

3 张统计卡片横向排列：

1. **加密状态**
   - 图标：🔒 已加密 / ⚠️ 部分未加密
   - 数字：已加密 Key 数 / 总 Key 数
   - 状态色：全加密绿色，部分黄色

2. **需要轮换**
   - 图标：🔄
   - 数字：检测到需要轮换的 Key 数
   - 触发条件：401 错误率超过阈值

3. **今日请求**
   - 图标：📊
   - 数字：今日总请求量 + 错误率
   - 迷你趋势图

### 5.3 Key 列表

每行显示：
- **Provider**：名称 + 图标
- **Key 前缀**：`sk-****abcd`（默认遮掩）
- **加密状态**：🔒 加密图标（hover 显示加密方式）
- **健康指示器**：
  - 🟢 健康（成功率 > 95%）
  - 🟡 警告（成功率 80-95%）
  - 🔴 需轮换（401 错误率 > 10% 或持续失败）
- **使用量**：今日请求数 / 月请求配额
  - 进度条：绿色(<60%) → 黄色(60-85%) → 红色(>85%)
- **错误率**：最近 1 小时错误率
  - 正常：`0.2%` 灰色
  - 警告：`5.3%` 黄色高亮
  - 危险：`15.7%` 红色高亮 + 脉冲动画
- **最后使用**：相对时间
- **操作按钮**：
  - 🔄 轮换（主要操作，蓝紫渐变按钮）
  - ⏸️ 暂停
  - 🗑️ 删除

**需要轮换的 Key 特殊样式：**
- 整行左边框：红色发光条
- Key 前缀文字闪烁红色
- 右侧显示"建议轮换"标签
- 操作区"轮换"按钮变为红色高亮

### 5.4 一键轮换流程

点击"轮换"按钮后：

**Step 1 — 输入新 Key**
- Modal 弹出
- 显示当前 Key 前缀（用于确认）
- 输入新 Key（password 类型，可切换显示）
- "测试连接"按钮（验证新 Key 可用性）
- 测试结果区：延迟、支持模型、权限

**Step 2 — 确认切换**
- 对比卡片：旧 Key vs 新 Key
- 切换策略选择：
  - 立即切换（Atomic swap）
  - 渐进切换（灰度，新 Key 先接 10% 流量）
- 确认按钮

**Step 3 — 完成**
- 成功动画（绿色对勾 + 轻量粒子效果）
- 显示切换时间、旧 Key 状态（已停用 / 保留备用）
- 操作记录写入轮换日志

### 5.5 轮换日志

默认折叠，展开后显示时间线：
- 每条记录：时间 + Provider + 操作类型 + 操作人 + 结果
- 操作类型：手动轮换 / 自动轮换 / Key 添加 / Key 暂停
- 筛选：按 Provider / 操作类型 / 时间范围

### 5.6 加密状态详情

点击加密状态卡片展开详情：
- 加密方式：AES-256-GCM
- 密钥管理：环境变量 / Vercel KV Encrypted
- 未加密 Key 列表（如有）+ "一键加密" 按钮

### 5.7 移动端降级

- 概览卡片垂直堆叠
- Key 列表每项变为独立卡片，主信息前置
- 轮换 Modal 全屏
- 日志使用时间线卡片布局

## 6. 交互细节

### 6.1 过渡动画
- 节点状态变化：颜色渐变 300ms ease-out
- 路由切换：连线流动动画速度变化
- Key 状态变化：脉冲动画 2s 循环
- Tab 切换：fade + slide 200ms

### 6.2 空状态
- 首次进入路由 Tab：引导卡片 + "开始配置" 按钮
- 无 Key 时安全 Tab：插画 + "添加第一个 Key" 引导

### 6.3 错误状态
- 网络错误：玻璃态 Toast，自动消失
- 操作失败：Modal 内内联错误提示，不关闭 Modal
- 权限不足：全屏遮罩 + 重新认证引导

## 7. 可访问性

- 所有状态色配合图标/文字，不仅靠颜色区分
- 键盘导航：Tab 顺序遵循视觉顺序
- reactflow 画布：提供缩放百分比读数
- 屏幕阅读器：节点使用 aria-label 描述完整状态
